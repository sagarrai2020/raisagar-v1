My workflow

Over the years i've built many websites in many different topics. I've experimented many different techonologies to make website look unique, visually appealing, bug free and how to make website load fast and SEO/user friendly and how to manage a website for future updates and customizations. Based on my personal experience and years of experiments, i know how to build a website to make everybody happy and that is "build everything from scratch to the exact needs of clients". Here is the details of how i build my websites from scratch.

  • Gathering Information

    First of all i'll gather relavant information from the client so that i can identify the client's needs, purpose of the website, target audience and business goals.

  • Design

    With all the information that has been gathered, i'll write a short script about website's purpose and goals and required functionalities then i'll create a prototype design using "Adobe XD", i'll pick the custom fonts, colors etc accroding to the type of website.

  • Front End Development

    Following the pre build prototype design, I'll write SEO friendly semantic HTML5 markup, I'll use Bootstrap to build beautiful, user friendly and cross browser compatible responsive layout means the website will render beautifully on any device and any browser, i'll use "npm", "Webpack" and "SASS" to customize Bootstrap and to write cleaner and moduler css/javascript code, i'll use "Git/Gihub" to esily maintain website's version and source code files and future updates and customizations.

  • Back End Development

    For dynamic user content driven websites i'll use "Django" and "Postgresql" to programme Database, an admin panel, user accounts, Interaction system between users, authentication and authorization system. Usually this type of websites has security concerns and i have deep understanding of security holes and how to prevent them and Django is the most secure Back-End technology i know.

  • SEO

    SEO is about increasing your website's visibility in the organic search results of major search engines such as google. I have extensive background in SEO, basically everything in SEO revolves around keywords, page title, heading and content, i know how to put these together logically in the right place for all of my websites.

  • Testing

    After the completion of website development, i'll launch the production ready website's folder from my computer to my production live server for testing purpose to test it's functionality, device and browser compatibility, load speed and performance in the real life enviroment to insure that the website is bug free and ready to deliver to the client.

  • Delivery

    I'll send the website link to the client. Client will check the website, if they ask me to change/add little things here and there, i'll change/add it and send the link again to the client. When the client is 100% setisfied, i'll withdraw the website from my server and deliver to the client and they'll host the website on their own server. If they ask me to handle domain name and hosting as well, they'll have to pay extra for the domain name(example.com) and hosting plan according to domain name and hosting type and every year for that subscription.

  • Support

    I am 100% responsible for any errors of my websites and i am 24/7 available to fix the errors. If clients need further customization in the future i can help them to customize the website to meet their newly created business format, for the customization charges will be applied.
